## The cardinal equation (the whole point)

```math
\begin{bmatrix} M & C_q^{\mathsf T} \\ C_q & 0 \end{bmatrix}
\begin{bmatrix} \ddot{q} \\ \lambda \end{bmatrix}
=
\begin{bmatrix} Q_e \\ Q_d \end{bmatrix}
```

Top row = Newton's law + joint reaction forces; bottom row = accelerations must
keep the joints closed. Solve each instant for `q̈` and `λ`; integrate.

## Concepts worked through (checklist + one-line recall)

- [x] **Why computational methods exist** — turn EOM *derivation* into a general
  algorithm; the principle (D'Alembert/Lagrange) is applied **once** to make the
  augmented template, then reused for every system. (NOTES preface)
- [x] **The layer map** — principle → governing eq → *formulation* → model →
  scheme. Augmented is one *formulation*; siblings: embedded/minimal, recursive
  O(n), Kane, Hamiltonian, penalty.
- [x] **`r = R + A(θ)ū`** — local→global: describe locally, orient (`A`), place
  (`+R`). `ū = [0, ±h]` because we align local `Yⁱ` along the rod.
- [x] **Reference point at COM** — makes `M` constant & diagonal (decouples
  translation/rotation). A "free" modeling choice.
- [x] **Constraints `C(q)=0`** — canonical revolute `C = r_i^P − r_j^P = 0`;
  ground pin is the `c = 0` special case. `C` = the **violation/residual** vector.
- [x] **Jacobian `C_q`** — same first-derivative/linearization object I know;
  it's `∂(constraint violation)/∂(coordinates)`. Rows = normals to the constraint
  surface; null space = allowed motions (the 3 DOF).
- [x] **Dependent vs independent** — 3 independent (angles, the true DOF) +
  6 dependent (Cartesian COMs, forced by joints). `C_qd` = square 6×6 block,
  invert it to recover dependent position/velocity.
- [x] **Newton–Raphson position step** `Δq = −C_qd⁻¹ C` — first-order Taylor,
  iterate to `ε`. *One step here* because angles fixed ⇒ constraints linear in
  the Cartesian unknowns.
- [x] **The time loop (a–e)** — recover dependent positions → dependent
  velocities → augmented solve (accel + λ) → store → RK4 advances angles.
  Position/velocity are pure kinematics ("free"); **acceleration is where forces
  enter**.
- [x] **`Q_d` (quadratic-velocity vector)** — `= −Ċ_q q̇`; the **velocity-induced**
  accelerations (centripetal + Coriolis). Tangential is NOT in it (that's
  accel-driven, lives in `C_q q̈`). This model = pure centripetal (no sliding ⇒
  no Coriolis). `C_q q̈ = Q_d` is a *matching condition* (like `F = ma`), not the
  definition of `Q_d`.
- [x] **`λ` (Lagrange multipliers)** — the joint **reaction forces**; "whatever it
  takes" (string-tension intuition). Units = **Newtons** here (length
  constraints). `C_qᵀ` gives directions, `λ` gives magnitudes.
- [x] **Why `C_qᵀ` is transposed** — same coupling read two ways: `C_q` =
  motion→constraint, `C_qᵀ` = constraint-force→coordinate. Force is the dual of
  motion (paired by work), so the force map is the transpose. Rows of `C_q` =
  constraint normals; the force is a `λ`-weighted sum of them.
- [x] **Real tools** — Drake/MuJoCo/Pinocchio and JPL's DARTS use **recursive
  O(n)** (minimal coords) for serial chains; the **augmented/`λ`** formulation
  (this repo) is used for **closed loops & contact**, and in tools like MSC Adams.
  NASA: arms, deployment (JWST), rover landing, flexible appendages (Shabana's
  flexible-MBD specialty).

## Analogies that made it click (don't lose these)

- **`λ` = string tension** holding a hanging weight: not chosen, *forced* to be
  "whatever keeps the constraint satisfied."
- **`Q_d` = car on a circular track at constant speed**: still accelerating
  (centripetally) even with zero tangential accel — that's why `C_q q̈ = Q_d`.
- **`C_q q̈ = Q_d` vs `F = ma`**: `Q_d` is *computed from velocities* and then
  *constrains* the accelerations, exactly like `F` is computed from force laws and
  `ma` must match — never the reverse.
- **`C_qᵀ` = mechanical advantage / gears**: the matrix relating displacements is
  the transpose of the one relating forces (principle of virtual work).

## Open questions / what to study next

1. **Recursive O(n) (Featherstone) formulation** — the minimal-coordinate sibling
   that Drake/JPL actually use. Understand why it's `O(n)` and how it avoids the
   augmented matrix for serial chains.
2. **3D / spatial dynamics** — `A(θ)` → quaternions/Euler params; 3×3 inertia.
3. **Constraint stabilization (Baumgarte) & DAE index** — how real solvers stop
   constraint drift (this repo re-projects every step instead).
4. **Flexible multibody** (Shabana's later chapters: floating frame of reference,
   ANCF) — the deformable-body case; directly relevant to spacecraft appendages.
5. **Hands-on Drake/Pinocchio** — build the same pendulum, compare; see where `λ`
   appears (loops/contact) vs minimal coords (the tree).
6. **Add a prismatic/slider joint** to this repo to make a real **Coriolis** term
   appear in `Q_d` and watch `−Ċ_q q̇` generate it automatically.
